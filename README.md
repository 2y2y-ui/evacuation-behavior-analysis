# Evacuation Behavior Analysis AI

災害時の避難行動を対象に、移動データから避難遅延リスクと混雑経路を検出するAIプロトタイプです。

このリポジトリでは、個人情報を含む実際のGPSログを使わず、仮想都市の避難行動データを生成します。ただし、データ形式は実社会の移動ログに近い形にそろえています。

```text
user_id, timestamp, location, latitude, longitude, action, speed, source
```

そのため、将来的にはスマートフォンGPS、人流データ、避難所オープンデータ、道路データなどに差し替えることで、実地域の避難支援に応用できます。

## 目的

- 避難が遅れやすい人・地域の特徴を発見する
- 混雑しやすい避難経路を可視化する
- 避難完了時間を予測する
- 実社会データへ接続しやすいデータ構造を作る

## 使用するAIアルゴリズム

- K-meansクラスタリング: 避難者の行動パターンを分類
- PCA: 多次元の避難特徴量を2次元に可視化
- 線形回帰: 避難完了時間を予測
- ニューラルネットワーク: 避難遅延リスクを分類

## セットアップ

```bash
pip install -r requirements.txt
```

## 実行方法

コマンドラインで分析を実行する場合:

```bash
python3 src/evacuation_simulator.py
```

StreamlitのWebアプリとして起動する場合:

```bash
streamlit run app.py
```

実行すると、ローカル環境の `data/` と `outputs/` にCSVと図が生成されます。生成物は再現可能なため、リポジトリには含めません。

## 生成されるファイル

- `data/evacuation_movements.csv`: 課題指定形式に近い移動ログ
- `data/evacuee_features.csv`: 避難者ごとの分析用特徴量
- `outputs/pca_clusters.png`: PCAによるクラスタ可視化
- `outputs/evacuation_map.png`: 仮想都市上の避難経路
- `outputs/delay_prediction.png`: 避難時間の実測値と予測値
- `outputs/route_congestion.png`: 混雑経路ランキング
- `outputs/summary_metrics.csv`: モデル評価指標

## 結果の例

現在のサンプルデータでは、以下のような結果が得られます。

- 線形回帰の平均絶対誤差: 約3.435分
- 線形回帰の決定係数R2: 約0.894
- ニューラルネットワークの遅延リスク分類精度: 約0.867
- 遅延リスク判定のしきい値: 避難完了時間52.4分以上

これにより、出発遅れ、避難距離、平均速度、経路混雑度が避難完了時間に影響することを確認できます。

## 実社会データへの応用

このプロトタイプは、次の公開・実務データと接続できます。

- 国土数値情報の避難施設データ
- OpenStreetMapの道路データ
- 人口メッシュデータ
- 自治体の避難訓練ログ
- 匿名化スマートフォンGPS・人流データ
- 洪水・津波・土砂災害ハザードマップ

本プロトタイプでは仮想データを使いますが、緯度経度、時刻、移動速度、滞在・移動状態を共通形式で扱うため、入力CSVを差し替えるだけで実地域への適用を検討できます。

## 開発フロー

このリポジトリでは、`main` に直接コミットせず、機能ごとにブランチを分けてPull Requestを作成します。

- 作業ブランチ: `feature/evacuation-behavior-analysis`
- Pull Request: 実装内容、検証方法、データの扱いを明記する
- 実データや個人情報はコミットしない
- 生成データを更新した場合は、再現コマンドも確認する

## データ利用上の注意

実際の災害時の個人移動ログはプライバシー保護のため公開利用が難しいです。そこで本研究では、仮想的な避難移動データを生成し、AIアルゴリズムによって避難遅延と混雑経路を分析します。これは、実社会データに接続する前段階の安全なプロトタイプとして位置づけられます。
